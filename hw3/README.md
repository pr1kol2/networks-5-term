# HW 3

Опишем дамп в третьем случае (оба клиента за NAT, каждый за своим). Точка захвата — линк между NAT3–Switch3.

1–2. `198.18.0.21:50001 → 198.18.0.10:40000 (UDP)` — клиент PC5 не знает как достичь PC6, обращается за помощью к rendezvous-серверу и  шлёт ему **REGISTER**.

3-3. `198.18.0.10:40000 → 198.18.0.21:50001 (UDP)` — rendezvous-сервер возвращает PC5 ответ **PEER …** с публичным и приватным адресами PC6. Далее UDP-трафик идет напрямую между клиентами.

4–5. PC5 отправляет пинги и на публичный, и на приватный адрес PC6. Как только от какого-то адреса приходит валидный ответ — дальше общение идёт через него.

6–9. Прямой p2p-чат без участия rendezvous ( "hello!", "hi", "uhmmmm", "bye").
